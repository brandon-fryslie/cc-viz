## Architecture Notes

### One Source of Truth
- ExtractSnippet() in service/snippet.go is the single snippet extraction implementation
- All search methods use this same function with consistent parameters (fullText, query, 100 chars context)
- No duplicate snippet logic across methods

### Return Type Pattern
- All search methods now return lightweight "SearchResult" types
- These types contain only what the frontend needs: ID, name, snippet, highlights, match count
- Full entity types (Extension, Todo, Plan) are no longer returned for search
- This follows the guideline: "One type per behavior" - search results are different from entity CRUD

### Highlight Offset Calculation
- Backend calculates byte offsets using UTF-8 rune-aware logic in ExtractSnippet
- Offsets are relative to the snippet string, not the full text
- Frontend can directly use these offsets with substring() operations
- No client-side text processing or guessing required
