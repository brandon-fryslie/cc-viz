## Sprint Goal

## Scope

**Deliverables**:
1. FTS5 extension search table and replace basic keyword matching

### P2: Backend - Unified Search Endpoint

**Acceptance Criteria**:
- [ ] Endpoint: `GET /api/v2/search?q={query}&types={types}&limit={limit}` implemented
- [ ] `types` parameter accepts comma-separated list: requests, conversations, extensions, todos, plans
- [ ] Default includes all types if `types` not specified
- [ ] Response format: unified result with section for each data type
- [ ] Response structure:
  ```json
  {
    "query": "search term",
    "results": {
      "requests": { "total": 10, "items": [...], "limit": 10, "offset": 0 },
      "conversations": { "total": 5, "items": [...], "limit": 10, "offset": 0 },
      "extensions": { "total": 3, "items": [...], "limit": 10, "offset": 0 },
      "todos": { "total": 2, "items": [...], "limit": 10, "offset": 0 },
      "plans": { "total": 1, "items": [...], "limit": 10, "offset": 0 }
    }
  }
  ```
- [ ] Each section returns: items list, total count, limit, offset
- [ ] Limit parameter applies per-section (each gets limit results)
- [ ] Performance: < 500ms for full cross-type search

**Technical Notes**:
- Handler function: `unifiedSearch()` in data_handler.go
- Call search methods in parallel for each type (for speed)
- Use goroutines with WaitGroup to parallelize
- Result struct: `UnifiedSearchResults` with typed sections
- Handle errors per-section (one type failing doesn't block others)
- Limit default: 10 per section, configurable

**Dependencies**:
- P0: Extension search must work
- P1: Todos and plans search must work
- Sprint 1 (Request search) must be complete

**Risks**:
- Parallel search with many results could be slow
- Mitigation: Cap results per type, use reasonable defaults
- Risk level: LOW

## Technical Architecture

### FTS5 Table Schemas

```sql
-- Extensions
CREATE VIRTUAL TABLE IF NOT EXISTS extensions_fts USING fts5(
  extension_id UNINDEXED,
  type,
  name,
  description,
  metadata_text,
  tokenize = 'porter unicode61'
);

-- Todos
CREATE VIRTUAL TABLE IF NOT EXISTS todos_fts USING fts5(
  todo_id UNINDEXED,
  content,
  status UNINDEXED,
  tags,
  tokenize = 'porter unicode61'
);

-- Plans
CREATE VIRTUAL TABLE IF NOT EXISTS plans_fts USING fts5(
  plan_id UNINDEXED,
  title,
  goal,
  content,
  status UNINDEXED,
  tokenize = 'porter unicode61'
);
```

### Data Flow (Unified Search)

```
Frontend: GET /api/v2/search?q=term&types=requests,conversations
  ↓
unifiedSearch() handler
  ├→ SearchRequests() [parallel]
  ├→ SearchConversations() [parallel]
  ├→ SearchExtensions() [parallel]
  ├→ SearchTodos() [parallel]
  └→ SearchPlans() [parallel]
  ↓
Merge results into UnifiedSearchResults
  ↓
Response with all sections
```
