## Sprint Goal

Implement full-text search for requests with hybrid indexing (both structured fields and optional full body content), enabling users to discover requests by endpoint, model, tools used, and response content.

## Scope

**Deliverables**:
1. FTS5 request search table and indexing pipeline
2. Backend search endpoint with snippet context
3. Frontend request search UI and API integration
4. Performance validation and benchmarks

## Work Items

### P0: Backend - FTS5 Indexing for Requests

## Technical Architecture

### Request FTS Table Schema

```sql
CREATE VIRTUAL TABLE IF NOT EXISTS requests_fts USING fts5(
  request_id UNINDEXED,
  timestamp UNINDEXED,
  method,
  endpoint,
  model,
  provider,
  tool_names,
  response_text,
  request_body_text,           -- Optional, for full-body search
  tokenize = 'porter unicode61'
);
```

### Data Flow

```
1. SaveRequest(requestLog)
   ↓
2. extractRequestText(requestLog)
   → Returns: method, endpoint, model, provider, tool_names, response_text, request_body_text
   ↓
3. IndexRequest (INSERT into requests_fts)
   ↓
4. Frontend: useSearchRequests(query, model)
   ↓
5. GET /api/v2/requests/search?q=...&model=...
   ↓
6. SearchRequests() (FTS query with OR logic)
   ↓
7. Extract snippets (60 chars context)
   ↓
8. Return paginated results with headers
```
